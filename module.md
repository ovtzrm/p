**Модули**
-

- Модулями называют механизм который разбивает тысячи строк на части.
Модуль(Modile) - это простой ффайл содержащий код на Puthone 

- каждый модуль может содержать переменные, классы и функции.

- кроме того, в модулях может находиться исполненный код.

- Имя модуля соотвесвует имени файла 

- импонтируется **без** расширений

**Пакет(package)**
-
- поделенный код может быть сгрупирован по схожей тематике файлов. Или если вынести часть модулей из проекта, чтобы их групировать в других проектах. 

- Пакет это набор взаимосвязанных модулей (при этом сам является модулем) предназначенных для решения задач определенного класса. Это способ структурирования данных. 
- Пакет - папка, содержащая в себе несколько модулей и другие пакеты. именем пакета является названием каталога. Он может быть импонтирован и может многократно использоваться.

- существуют Стандартные библиотеки, Сторонние модули, Пользовательские модули.

**Стандартные библеотеки**


os - для работы с операционной системой, получаемые версии с дргой информацией текущей ОС
random - модуль предоставляет функции для псевослучайных чисел
re - работает с регулярными выражениями 
collestion - предоставляет специализированные контейнеры для работы с данными.

**Сторонние модули**

Это модули и пакеты которые не входят в диструктив Python, но могут быть установленны из каталога с помощью менеджера pip.
pyPI это главный каталог библеотек в Puthon. Он содержит более 200000 пакетов. 

- requests (HTTP - для Питона )
- numpy(вычесления с многомерными массивами)
- flask (пакет для создания веб-приложений)

При импорте модуля Python выполняет весь код в нём.
При импорте пакета Puthon выполняет код в файле пакета _init_.py если он имеется.
Все обьекты определенные в модуле или _init_.py, становятся поступны импортирующему коду

```py
import <модуль/пакет_1> [ as <псевдоним_1> ] [, модуль/пакет_2 as <псевдоним_2> ]...
```
- После ключевого слова inport указывается название модуля. одной конструкцией можно подключить несколько модулей, но **рекомендуется так не делать.** После импонтирования модуля его название становится переменной, через которую можно получить доступ к атрибутам модуля. если название модуля слишком длинное, для него можно создать псевдоним с помощью слова **as**. При использовании псевдонима, доступ ко ввсем атрибутам модуля осуществялется только с помощью переменной-псевдонима.
